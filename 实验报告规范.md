# 软件工程实验报告规范

> 适用于 `24046523陆一睿软件工程.docx` 的撰写与排版

---

## 一、整体结构

报告采用章节制，每个实验独立成章（Heading 1）：

```
封面（学校/课程/学号/姓名/日期）
目录（含超链接，指向各章 Heading 1 书签）
实验一：需求分析与最小可用产品
实验二：模块化设计
实验三：测试与验收
实验四：Git 协作与版本控制
实验五：迭代开发与变更管理
实验六：持续集成
实验七：发布与运维
实验八：AI Harness 工程
附录（可选）
```

## 二、每个实验的通用章节

| 编号 | 章节 | 内容 |
|------|------|------|
| X.1 | 实验目标 | 摘录大纲对应 SWEBOK 领域和教学目标 |
| X.2 | 核心代码 | 关键代码片段 + 说明文字 |
| X.3 | 运行结果 | 截图 + 运行说明 |
| X.4 | 实验结果 | 对应大纲评分点逐项自评 |
| X.5 | 课后反思 | 回答大纲"课后思考"问题 |

## 三、各实验特有内容

| 实验 | 特有章节 |
|------|----------|
| 实验一 | 用户采访记录、用户故事与验收标准、验收标准对照表 |
| 实验二 | 模块关系图（ASCII or 图片）、重构前后对比表、各模块职责说明 |
| 实验三 | 需求与测试用例对应表、Bug 修复记录 |
| 实验四 | Git 分支图、PR 截图、合并冲突说明 |
| 实验五 | 变更请求表（用户原话→验收标准→工作量）、变更选择理由、CHANGELOG |
| 实验六 | CI 配置说明、失败→修复流程图、CI 截图 |
| 实验七 | 用户手册（USER_GUIDE）、Bug 报告→复现→修复流程 |
| 实验八 | Harness 架构图、Eval 结果、800 字 AI 工程反思 |

## 四、格式要求

- 封面：按学校模板，含课程名、学号、姓名、日期
- 目录：含超链接（Heading 1 → 书签），字体黑色
- 标题：使用 docx Heading 1~4 样式（导航窗格可见）
- 正文：宋体/黑体，5号，1.5倍行距
- 代码：等宽字体（Consolas），灰底区块
- 全文字体黑色，无彩色字体
- 截图清晰，带编号和图题
