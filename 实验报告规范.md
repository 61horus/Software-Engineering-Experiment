## 〇、交付物分界：仓库 vs 报告

大纲列出的交付物分属两处，互不重叠：

| 去向 | 内容 | 格式 |
|------|------|------|
| **Gitee 仓库** | 源码、测试、配置、Markdown 文档 | 文本（可 diff） |
| **docx 报告** | 截图、运行结果展示、封面目录、反思 | 二进制/排版 |

具体分界：

| 大纲交付物 | 仓库中有？ | 报告中呈现？ |
|-----------|:---:|:---:|
| main.py / task_model.py 等源码 | ✅ 直接提交 | ❌ 仅贴关键片段 |
| tasks.json 示例 | ✅ `tasks_example.json` | ❌ |
| README.md（用户故事） | ✅ `campus_task/README.md` | ❌ 可引用 |
| pytest 测试用例 | ✅ `tests/` | ❌ 仅贴用例表 |
| 模块关系图 | ✅ `实验交付物/实验二_模块关系图.md` | ✅ 报告中粘贴图片版 |
| 变更请求表 | ✅ `实验交付物/实验五_变更请求表.md` | ✅ 报告中以表格呈现 |
| CHANGELOG.md | ✅ `CHANGELOG.md` | ❌ |
| CI 配置文件 | ✅ `.github/workflows/test.yml` | ❌ 仅贴说明 |
| 用户手册 | ✅ `campus_task/USER_GUIDE.md` | ❌ |
| Bug 报告 | ✅ `实验交付物/实验七_Bug报告与修复.md` | ✅ 报告中呈现 |
| Harness 架构图 | ✅ `实验交付物/实验八_Harness架构图与反思.md` | ✅ 报告中粘贴图片版 |
| 运行截图 | ❌ 仓库不上传（已 gitignore） | ✅ 报告中粘贴 |
| 课后反思 | ✅ 各 .md 文件中有文字版 | ✅ 报告中正式呈现 |

> 原则：文本留仓库，截图进报告。仓库是工程证据，报告是评审材料。

## 一、整体结构

报告采用章节制，每个实验独立成章（Heading 1）：

```
封面（学校/课程/学号/姓名/日期）
目录（含超链接，指向各章 Heading 1 书签）
实验一：需求分析与最小可用产品
实验二：模块化设计
实验三：测试与验收
实验四：Git 协作与版本控制
实验五：迭代开发与变更管理
实验六：持续集成
实验七：发布与运维
实验八：AI Harness 工程
附录（可选）
```

## 四、格式要求

- 封面：按学校模板，含课程名、学号、姓名、日期
- 目录：含超链接（Heading 1 → 书签），字体黑色
- 标题：使用 docx Heading 1~4 样式（导航窗格可见）
- 正文：宋体/黑体，5号，1.5倍行距
- 代码：等宽字体（Consolas），灰底区块
- 全文字体黑色，无彩色字体
- 截图清晰，带编号和图题
