# 实验二 — 模块关系图

```
┌─────────────────────────────────────┐
│            main.py                   │
│        (命令行交互层)                 │
│  解析 sys.argv → 分发到 service      │
│  格式化输出结果                      │
└──────────────┬──────────────────────┘
               │ 调用
               ▼
┌─────────────────────────────────────┐
│          task_service.py             │
│         (业务逻辑层)                  │
│  add_new_task()  /  get_all_tasks()  │
│  mark_task_done()                    │
│  返回状态码给 main，不处理显示        │
└──────┬──────────────────┬───────────┘
       │ 调用              │ 调用
       ▼                   ▼
┌──────────────┐   ┌──────────────────┐
│ task_model   │   │  task_storage    │
│   (模型层)    │   │   (存储层)        │
│              │   │                  │
│ create_task()│   │ load_tasks()     │
│ get_next_id()│   │ save_tasks()     │
│              │   │ _init_example()  │
│ 纯函数       │   │ _read_json()     │
│ 无依赖       │   │ 含容错处理        │
└──────────────┘   └──────────────────┘
```

## 各模块职责

| 模块 | 职责 | 依赖 |
|------|------|------|
| main.py | 命令行参数解析、输出格式化 | task_service |
| task_service.py | 业务逻辑：增/查/改 | task_model, task_storage |
| task_model.py | 数据结构定义、编号计算 | 无 |
| task_storage.py | JSON读写、容错、示例初始化 | 无（仅os/json） |
